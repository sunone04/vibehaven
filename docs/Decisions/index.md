# 决策记录

团队决策记录（ADR — Architecture Decision Records）。Agent 在此理解"为什么这样决策"，避免重复讨论已定事项。

## 决策清单

> 按时间倒序排列。

| 编号 | 标题 | 状态 | 日期 |
|---|---|---|---|
| _待填写_ | | | |

## 决策模板

> 新增决策时复制此模板到独立文件（如 `ADR-001-使用某技术.md`）。

```
# ADR-XXX — 决策标题

## 状态
提议 / 已接受 / 已废弃 / 已替代

## 日期
YYYY-MM-DD

## 背景
为什么要做这个决策？面临什么问题？

## 决策
我们决定什么？

## 备选方案
考虑过哪些方案？为什么没选？

## 后果
这个决策带来什么正面和负面影响？
```

## 维护规则

- 决策一旦接受不可删除，只能标记为"已废弃"或"已替代"
- 替代旧决策时，新决策须引用旧决策编号
- 重大架构变更须先记录决策再执行
